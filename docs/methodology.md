# Penetration Testing Methodology

## Overview

A structured methodology was followed to ensure systematic testing and documentation.

## Phase 1 – Reconnaissance

The objective of reconnaissance is to collect information about the target system.

Activities:
- Host discovery
- Port scanning
- Service identification
- Technology fingerprinting

## Phase 2 – Enumeration

Enumeration helps identify:
- Directories
- Parameters
- Forms
- Hidden pages
- Authentication mechanisms

## Phase 3 – Vulnerability Assessment

Testing was performed against:
- SQL Injection
- XSS
- CSRF
- Command Injection
- File Upload Vulnerabilities
- Authentication Weaknesses

## Phase 4 – Exploitation

Validated identified vulnerabilities through controlled exploitation.

## Phase 5 – Reporting

Documented:
- Risk level
- Impact
- Evidence
- Mitigation recommendations
